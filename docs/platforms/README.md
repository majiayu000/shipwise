# Platform Source Index

This directory is the source layer for launch decisions.

Before an agent posts, it should open the relevant platform page here and check the linked official source. If the official source changed, update the local page before using it.

| Platform | Local guide | Use when |
|---|---|---|
| GitHub | [github.md](github.md) | Every open-source project |
| Package registries | [package-registries.md](package-registries.md) | CLI, library, SDK, binary tools |
| Hacker News | [hacker-news.md](hacker-news.md) | Runnable technical projects and deep technical posts |
| Reddit | [reddit.md](reddit.md) | Niche community feedback |
| Product Hunt | [product-hunt.md](product-hunt.md) | Productized apps/tools with visual assets |
| X | [x.md](x.md) | Owned distribution, updates, demos, social proof |
| Blog/newsletter | [blog-newsletter.md](blog-newsletter.md) | Durable explanation and SEO |
| Chinese communities | [chinese-communities.md](chinese-communities.md) | Chinese developer audience and localized adoption |

## Common Rule

Use fewer channels and better adaptation. A launch to three well-chosen platforms with strong proof is better than ten generic posts.
