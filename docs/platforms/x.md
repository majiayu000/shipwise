# X Source Guide

X is an owned distribution and conversation channel. It is useful for demos, release notes, build stories, and repeated learning, but it should not become spam.

## Official Sources

- X automation rules: https://help.x.com/en/rules-and-policies/x-automation
- X platform manipulation and spam policy: https://help.x.com/rules-and-policies/platform-manipulation

## Use When

- The project owner has an existing audience or wants to build one.
- There is a visual demo, surprising result, benchmark, or build story.
- The launch needs ongoing updates, not just one announcement.

## Agent Checklist

- [ ] Thread has one clear hook.
- [ ] Demo/proof appears early.
- [ ] Repo or product link is included.
- [ ] Limitations are honest if relevant.
- [ ] No duplicated unsolicited replies.
- [ ] No automated DMs or engagement loops.
- [ ] Comments and useful replies are captured after launch.

## Do Not

- Do not mass-reply to unrelated accounts.
- Do not automate reposting or engagement without checking current X rules.
- Do not present AI-generated claims as measured facts.
